PATHFORGE — branching map builder

Build Slay-the-Spire / Inscryption-style branching maps: place nodes, join them with roads, rails, dirt, brick, stairs or ladders, paint the board with a terrain, drop in an avatar and tap nodes to walk it round the map — then export clean JSON (or a paste-in JS block) for your game. Everything autosaves in this browser. ＋ New map adds another tab.

Map

Paths drawn is the surface every new link and the generator use — tap an existing link to restyle just that one. Floors are the rungs of the map; nodes keep their spots and re-attach to the nearest lane when it changes.

Generator

Walks Paths random routes up the floors (paths can share nodes but never cross, same as Slay the Spire), then deals out node types by the sliders. Links come out in the map's Paths drawn surface. Generating replaces the current nodes — Undo gets the old map back.
Fills the board with a full R×C lattice — every node joined to its neighbours (add the diagonals for 8-way). Rows become floors, so it still walks bottom→top / left→right and exports the same way. Great as a game-board, dungeon grid, or overworld.

Avatar

In ▶ Play test, tap any node and the avatar walks the paths to it — forwards along the arrows if it can, backtracking if it must. In Select mode, pick a node and hit Send avatar here. The avatar and its position are saved with the map.

Selected node

↑ change a single node's type here — Enemy, Elite, Merchant, Rest…
Tap a node on the board to edit it here. While it's selected, drag it with one finger (the board won't scroll), or nudge it with the arrow keys. Tap a link to select the link instead.

Export & import

JSON gives you {nodes:[…], edges:[[from,to,surface?],…]} — ids are stable strings, edges always point start → boss, and the surface is only written when it isn't the plain trail. The JS block is the same data as a paste-in const. Import accepts a single map or a whole {maps:{…}} bundle; the export box is editable, so paste JSON in and hit Import from box.
